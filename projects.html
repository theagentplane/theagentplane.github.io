Chronicle

★ 2 stars · Python · MIT License
View repo ↗

Chronicle captures the execution DNA of your agents. It records granular decision states — model versions, retrieval snapshots, tool schemas, and environment context — enabling accurate traces that let you navigate your agentic terrain with absolute observability.

How it works

Chronicle implements two complementary systems, built as a side execution graph that sits alongside your agent without modifying its framework topology:

  • Agent Data Recorder ("The Envelope") — captures flight data at decision boundaries: every LLM call, tool invocation, or routing decision gets serialized into an immutable, append-only Envelope with contextual metadata, input state, action/result, and graph linkage for retries.
  • Verification Test Bench — a two-layer testing model. Layer 1 replays static envelope fixtures for deterministic structural assertions without ever calling an LLM API. Layer 2 uses LLM-as-judge evaluation to validate generation quality — grounding, safety, refusal — where bitwise equality doesn't apply.

Production incidents get committed under fixtures/traces/ as permanent regression tests — including shipped demos for a $9.8M refund misfire, a currency-mismatch invoice, and an oversized trade, each with a gated fix verified via cut-point replay.

tokenops

★ 1 star · Reference Architecture
View repo ↗

An operational field guide and reference architecture for treating tokens as a first-class infrastructure resource. Built for the AI Engineer World's Fair 2026 (June 29 – July 2, San Francisco).

The thesis

Traditional cloud cost controls — budgets, quotas, rate limits — are built for deterministic, bounded workloads where a request costs roughly the same every time. Agentic systems break that assumption: token consumption is stochastic, varies with model reasoning depth, retries, and tool-calling chains, and can spike unpredictably within a single session. tokenops proposes treating tokens the way infrastructure teams already treat compute and storage — with first-class observability, budgeting primitives, and governance built for variance rather than fighting it.
